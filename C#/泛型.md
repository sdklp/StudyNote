C# **泛型**（Generics）允许你编写适用于各种数据类型的通用代码，同时保持严格的类型安全性和高性能。它将类型的具体指定延迟到实例化或调用时进行，避免了代码重复和装箱/拆箱操作。 

核心优势

- **代码复用**：一份逻辑可应用于多种类型。

- **类型安全**：编译时检查类型，避免运行时强制转换错误。

- **高性能**：避免值类型的装箱与拆箱。

  

1. 泛型类与泛型接口

通过在名称后添加 `<T>`（或其它占位符）来定义，`T` 代表客户端指定的类型

```
// 泛型类
public class GenericList<T>
{
    private List<T> _items = new List<T>();
    
    public void Add(T item) => _items.Add(item);
    public T Get(int index) => _items[index];
}
```

2. 泛型方法

方法可以独立于类定义泛型，通常参数类型可由编译器自动推断。

```
// 泛型方法
public void Swap<T>(ref T a, ref T b)
{
    T temp = a;
    a = b;
    b = temp;
}

// 调用时编译器会自动推断 T 为 int
int x = 1, y = 2;
Swap(ref x, ref y); 
```

3. 类型参数约束 (Constraints)

使用 `where` 关键字可以限制泛型能接受的类型，从而调用特定类型的方法。 

- `where T : class`：类型参数必须是引用类型，包括任何类、接口、委托或数组类型；
- `where T : struct`：类型参数必须是值类型。可以指定除NULLable以外的任何值 类型
- `where T : new()`：必须包含无参数的公共构造函数。当与其他约束一起使用时，new()约束必须最后指定。
- `where T : 接口名`：类型参数必须是定的接口或实现指定的接口。可以指定多个接口约束。约束接口也可以是泛型的。
- `where T : 类名`：必须继承自指定的基类

```
// 约束 T 必须实现 IComparable 接口
public T GetMax<T>(T a, T b) where T : IComparable<T>
{
    return a.CompareTo(b) > 0 ? a : b;
}

```

4. 协变和逆变 (Covariance and Contravariance)

允许将泛型接口或委托赋给派生程度更大或更小的变量。

- **协变 (`out`)**：允许使用比原始指定的派生程度更大的类型（只能作为返回值）。
- **逆变 (`in`)**：允许使用比原始指定的派生程度更小的类型（只能作为输入参数）。

```
// 协变示例，假设 Dog 继承自 Animal
IEnumerable<Dog> dogs = new List<Dog>();
IEnumerable<Animal> animals = dogs; 

```
